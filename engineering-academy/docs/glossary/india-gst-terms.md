# India GST Terms

If you're not familiar with India's GST system, this vocabulary will come up constantly in [API → GST](/api/gst) and [Backend → Services: NIC API](/backend/services-nic-api) — worth skimming even if you never touch tax code directly, since it explains *why* certain fields exist on invoices at all.

| Term | Definition |
|---|---|
| **CESS** | An additional levy on top of GST for specific goods (e.g. luxury or sin goods); stored as a separate line item where applicable. |
| **CGST / SGST** | Central GST and State GST — for sales *within* the same state, tax is split roughly in half between these two, each going to a different government. |
| **E-Invoice / IRN** | A government-registered invoice: the seller submits invoice data to NIC, which returns an **IRN (Invoice Reference Number)** and a QR code — mandatory above certain turnover thresholds. See [buildIrnPayload](/backend/services-nic-api). |
| **EWB (E-Way Bill)** | A document required for transporting goods above a value threshold, generated via a similar NIC handshake to the e-invoice, containing transporter and vehicle details. |
| **GSTIN** | The 15-character GST Identification Number assigned to every registered business — validated in this codebase by `isValidGstin()`, which checks length, checksum, and structure. |
| **GSTR-1** | The monthly/quarterly return listing all *outward* (sales) supplies a business made — one of the export formats in [reports.ts](/api/gst). |
| **GSTR-2B** | An auto-drafted statement of *inward* (purchase) supplies, generated by the government from suppliers' GSTR-1 filings, used for reconciliation. |
| **GSTR-3B** | The monthly summary return of total tax liability and input tax credit claimed — the actual return used to determine tax payable. |
| **HSN Code** | Harmonized System of Nomenclature — an international product classification code; required on invoice line items to determine the correct GST rate. |
| **IGST** | Integrated GST — applied instead of CGST+SGST when a sale crosses state lines (interstate); the entire amount goes to the central government, which settles with states. |
| **Input Tax Credit (ITC)** | Tax paid on purchases that a business can offset against tax owed on its own sales — the core mechanism GSTR-2B reconciliation exists to verify. |
| **NIC** | National Informatics Centre — the Indian government body operating the IRN/EWB registration APIs that `nic-api.ts` integrates with. |
| **Reverse charge** | A mechanism where the *buyer*, not the seller, is liable to pay GST directly to the government — relevant for certain supplier categories. |
| **Sandbox mode** | NIC provides a separate test environment with its own credentials and endpoints; `nic-api.ts`'s three modes (mock/sandbox/production) map directly onto this government-provided staging tier. |
| **Tax invoice vs. bill of supply** | A tax invoice is issued when GST applies; a bill of supply is issued by GST-exempt or composition-scheme sellers who cannot charge GST at all. |
